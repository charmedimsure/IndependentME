Setting up IndependentME

Fifteen steps, about twenty minutes. Everything here is free.

Nothing you do reaches Karelynn until step 13. Up to that point you can get it wrong, redo it, and she never knows.

0 of 15 done
STEP 01Put the files in a folder

Download all six files I gave you. Make a folder called independentme somewhere you'll find it again, and arrange them exactly like this:

independentme/ ├── worker.js ├── wrangler.toml ├── schema.sql └── docs/ ├── index.html ← this is independentme.html, renamed └── care.html

The docs folder is one you create. Renaming independentme.html to index.html is what makes her web address short and clean instead of ending in a filename.

You can leave README.md in the top folder or delete it. It doesn't affect anything.

STEP 02Open a terminal in that folder

Windows: open the independentme folder in File Explorer, click the address bar at the top, type cmd, press Enter.

Mac: right-click the folder → Services → New Terminal at Folder.

A black or white text window opens. Type this and press Enter to confirm you're in the right place:

dir        (Windows)
ls         (Mac)
You should seeworker.js, wrangler.toml, schema.sql and docs listed. If you don't, you're in the wrong folder.
STEP 03Check you have Node

In that same terminal window:

node --version
You should seeA version number like v20.11.0. Anything v18 or higher is fine.
If it says "not recognized"Go to nodejs.org, download the LTS version, install it with all the defaults, then close the terminal and open a new one. It won't work in the old window.
STEP 04Make a Cloudflare account

Go to dash.cloudflare.com/sign-up and make a free account. Email and password, confirm the email, done. You don't need a domain and you won't be asked for a card.

Cloudflare is where the small piece of server code lives — the thing that lets her tablet and your four phones see the same information. The free tier is far more than this will ever use.
STEP 05Install wrangler and sign in

Back in the terminal:

npm install -g wrangler

That takes a minute and prints a lot of text. Warnings in yellow are normal. Then:

wrangler login

A browser window opens asking you to authorise it. Click Allow.

You should seeThe terminal says you're logged in successfully.
STEP 06Create the database
wrangler d1 create independentme-db

It prints a small block of text ending in a long database_id — a string of letters, numbers and dashes. Copy that ID and paste it here so the rest of this guide can use it:

Saved on this page only. Nothing is sent anywhere.
STEP 07Put the ID into wrangler.toml

Open wrangler.toml in Notepad or TextEdit. Find the last line:

database_id = "PASTE_THE_ID_WRANGLER_PRINTS_HERE"

Replace it with yours, keeping the quote marks:

database_id = "your-id-here"

Save the file and close it.

Watch outIf you're on Mac and used TextEdit, make sure it saved as plain text, not rich text.
STEP 08Build the tables
wrangler d1 execute independentme-db --remote --file=./schema.sql

It'll ask you to confirm. Type y and press Enter.

The one thing not to skipThat --remote is doing real work. Without it, the tables get built in a practice copy on your own computer, everything looks like it worked, and then nothing ever saves. If you're unsure whether you included it, just run the command again — running it twice is harmless.
You should seeThree or four commands executed successfully.

STEP 10Put the two apps online

On GitHub, make a new repository called independentme. Public is fine — the codes are what keep it private, not the repository setting.

Upload the folder contents. Easiest way if you'd rather not use git: on the empty repo page click uploading an existing file, then drag in worker.js, wrangler.toml, schema.sql, and the whole docs folder.

Then Settings → Pages. Under Source pick Deploy from a branch, branch main, folder /docs. Save.

Wait about a minute, then refresh that page — it'll show your address at the top.

You should seeOpening that address shows her app saying "Not set up yet." That's exactly right. Adding care.html to the end shows the dark caregiver app.
STEP 11Choose your two codes

Make these up now. Longer is better.

Goes on the tablet and all four phones.
Phones only, never the tablet. This is what allows changes.
Write them down somewhere that isn't a group chat. There's no password reset — if you lose these you'd start over, which costs about ten minutes.
STEP 12Set up your phone

On your phone, open your-pages-address/care.html. Go to the Setup tab and fill in all three: worker address, household code, caregiver key. Tap Save and connect.

You should seeThe dot at the top right turns green and says connected. Now check the Today tab — a list of tasks should be there. That means the database took the write. This is the moment it all becomes real.
If the dot stays redAlmost always a typo in the worker address. It needs the https:// and no slash on the end.

While you're here: add IndependentME to your phone's home screen so you're not hunting for a browser tab. Then set her name, and go through the task list — times, timers, what's daily versus weekly.

STEP 13Add the four of you

Install ntfy from the Play Store or App Store on each phone. It's free and needs no account. Each person picks a topic name — make them long and random, like nan-km-4k2p9 — and subscribes to it in the app.

In Setup → People, add yourself, Seth, Shelby and Nan. Give each one their topic and an emoji or photo. Then set who you are on this device.

Now use Copy an invite link for each of the other three. That link has everything pre-filled including their name, so they tap it once and they're set up.

Send those individuallyThe invite link contains the caregiver key. Person to person, not to a group thread.
Do the topics before the invite links. The link carries the setup as it exists when you generate it.
STEP 14Set up her tablet

On the tablet, in Chrome, open your-pages-address.

Press and hold the faint grey gear in the top right corner for about a second. The setup panel appears. Enter only the worker address and household code — the tablet never gets the caregiver key. Tap Save and connect.

You should seeHer whole day appears — weather at the top, tasks below, sun on the arc. If it's morning the screen is amber; evening, deep purple.

Then, in order:

  • Tap Turn on notifications and allow it
  • Chrome menu → Add to Home Screen, then open it from that icon
  • Settings → Display → Screen timeout → the longest option
  • Settings → Security → App pinning → turn on. Open IndependentME, tap the Recents button, tap the app's icon, choose Pin. Home and Recents now need a PIN.
  • Plug it in and leave it plugged in
STEP 15Test it before she relies on it

Do this yourself, on both devices, before you show her.

  • A task. On your phone set some task to two minutes from now. Wait. The tablet should take over the whole screen and say it out loud.
  • The timer. Start brushing teeth on the tablet. The ring should drain and it should check itself off at the end.
  • A message. Send one from the tablet to yourself. It should land in your care app, and your ntfy should buzz.
  • A reply. Answer from your phone. It should appear on the tablet and be read aloud.
  • Help. Trigger an alarm, tap Not yet until the help button appears, ask someone. Their phone should buzz. Tap I've got this and the tablet should say they're coming.
  • Offline. Turn the wifi off on the tablet. Her day should still be there and still be tappable. Turn it back on and your phone should catch up.
Once those all passYou're done. Anything that failed, tell me which one and what happened — that's usually enough for me to name the cause.
